# Prove v1.1 — Formatter + View

## Goal

Port `formatter.py` and the `prove view` command from the Python bootstrap
compiler to the self-hosted Prove compiler. After this release, `prove format`
and `prove view` work as native commands without Python.

**Depends on:** v1.0 (self-hosted compiler with lexer, parser, checker).

---

## Commands

### `prove format [path] [--check] [--stdin] [--md]`

| Flag | Behavior |
|------|----------|
| `path` | File or directory (default `.`) |
| `--check` | Exit 1 if any file would change; don't modify files |
| `--stdin` | Read from stdin, write formatted to stdout |
| `--md` | Also format `` ```prove `` fenced code blocks in `.md` files |

### `prove view <file>`

- Dump parsed AST in human-readable tree format
- Exclude span information from output (clean structural view)
- One declaration per tree, indented with 2 spaces

---

## Implementation

### Formatter Module

Port `formatter.py` (~840 lines Python) to a Prove `Formatter` module.

**Key components:**
- Indent tracking and emission
- Expression formatting with precedence-aware parenthesization
- Type expression formatting (generics, modifiers)
- Contract block formatting (ensures, requires, explain, terminates)
- Line-length-aware wrapping (configurable via `prove.toml` `[style]`)
- Type inference from symbol table (for `--stdin` mode, imports needed)

**Stdlib dependencies:** `Text` (string building), `Character` (whitespace
detection), `InputOutput` (file read/write).

The formatter needs the symbol table from the checker for type inference
during formatting. The pipeline is: lex → parse → check → format.

### View Module

Port AST dump logic from the `view` CLI command.

**Key components:**
- Recursive AST walker that produces indented text
- Type/expression/statement/declaration pretty-printing
- Span suppression (spans are internal, not useful for `view` output)

**Stdlib dependencies:** `Text` (string building), `InputOutput` (stdout).

### CLI Integration

Add to the self-hosted `Cli` module:

```prove
inputs cmd_format(path String, check Boolean, stdin Boolean, md Boolean)!
inputs cmd_view(path String)!
```

Argument parsing: extend the `main` dispatch to handle `format` and `view`
subcommands with their flags.

## Parity Confirmation

Formatter and view output must be **identical** between the Python and
self-hosted compilers. Any divergence is a bug.

### Automated Parity Tests

1. **Format every `.prv` file** in `tests/fixtures/` and `stdlib/` with both
   compilers. `diff` the output — must be identical.

2. **`prove format --check`** on the entire `prove/` source directory (the
   self-hosted compiler source) with both compilers — both must report the
   same files as needing changes (or none).

3. **`prove view`** on 10 representative files:
   - `stdlib/input_output.prv` (IO verbs, channels)
   - `stdlib/text.prv` (pure verbs, builder pattern)
   - `stdlib/table.prv` (binary type, generics)
   - `stdlib/character.prv` (simple pure functions)
   - `stdlib/parse.prv` (algebraic types, match)
   - A file with `explain` blocks
   - A file with `match` expressions
   - A file with string interpolation
   - A file with lambda expressions
   - A file with refinement types

   Compare AST dumps from both compilers — must be identical.

4. **Round-trip test:** format a file, then format the result again. Output
   must be identical (idempotency).

### Manual Spot Checks

- Format a file with `--md` flag, verify fenced Prove blocks are formatted
- Format via `--stdin`, verify stdout output matches file-based formatting
- `prove format --check` returns exit code 0 on already-formatted files
- `prove format --check` returns exit code 1 on unformatted files
