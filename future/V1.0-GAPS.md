## 1. Comptime Execution (v0.9.3 plan — incomplete)

The interpreter (`interpreter.py`, 368 lines) exists and is called by the optimizer for
constant-folding pure calls. However, full comptime block execution as described in docs
is incomplete.

- [x] `comptime` keyword parsed in lexer/parser
- [x] `comptime` expressions type-checked in checker (E410–E422 codes)
- [x] Tree-walking interpreter for pure constant expressions (literals, binary ops, unary ops, match)
- [x] `read()` function for file I/O in comptime
- [x] Optimizer calls interpreter for compile-time evaluation of pure calls with constant args
- [x] **Comptime blocks in user code** — `comptime` is now a valid expression in any position
      (function body var decls, etc.) by adding `TokenKind.COMPTIME` handling in `_parse_prefix()`.
      The emitter already handled `ComptimeExpr` via `_eval_comptime`. Evaluated and inlined
      as C constants at compile time.
- [ ] **Build dependency tracking** — files read via comptime `read()` should become build deps;
      not wired into builder
- [ ] **Comptime match for conditional compilation** — documented in `compiler.md` but no
      evidence of working in emitter

**Refs:** `docs/compiler.md` (Comptime section), `prove-py/archive/0.9.3-PLAN.md`,
`interpreter.py`, `optimizer.py:_ct_eval_pure_calls`

## 2. Linear Types & Ownership (v0.9.4/v0.9.7 plans — partial)

- [x] `Own` and `Mutable` type modifiers defined in `types.py`
- [x] Basic use-after-move detection via `_moved_vars` set in checker (E340)
- [x] `BorrowType` defined and used for compiler-inferred read-only parameter borrows
- [x] `_infer_param_borrows()` in checker detects read-only parameters
- [~] **Move tracking** — tracks moves through function call arguments (`_check_calls.py:432`),
      assignment (`checker.py:1535–1538`), nested call chains (`f(g(owned))`), pipe expressions
      (`x |> f`), and field paths; does NOT track:
  - [x] Assignment moves (`x = y` where y is Own)
  - [x] Complex expression moves (chained calls and pipe; added `_check_calls.py`)
  - [x] Field-path moves (partial moves of owned struct fields)
  - [x] Return-position moves — Own-typed identifiers returned from a function are now
        marked as moved in `_moved_vars` (`checker.py:_check_function`)
- [x] **Borrow inference** — `_infer_param_borrows()` is wired and active
      (`checker.py:840–845`); infers read-only parameters and wraps their types as
      `BorrowType` to avoid unnecessary copies

**Refs:** `docs/types.md` (Ownership Lite), `prove-py/archive/0.9.4-PLAN.md`,
`prove-py/archive/0.9.7-PLAN.md`, `checker.py:2115–2210`, `_check_contracts.py:430`

## 3. Memory Regions for Local Temporaries (v0.9.5/v0.9.7 plans — partial)

The region runtime exists and is initialized, but is NOT used for local temporary allocation.

- [x] `prove_region.c/h` runtime with `enter`/`exit`/`alloc` (133 lines total)
- [x] `_global_region` created in `prove_runtime.c` init
- [x] **Per-function region scoping** — `prove_region_enter/exit` emitted in ALL
      functions via `_emit_function()` (`c_emitter.py:595,619`), not just main
- [x] **`prove_region_alloc` for string and list temporaries** — string literals (`StringLit`,
      `TripleStringLit`, `RawStringLit`, `PathLit`) and list literals now use
      `prove_string_from_cstr_region` / `prove_list_new_region` when inside a function with
      escape analysis and not in return position. Return-position expressions use malloc so
      the value survives region exit. Other allocation (struct fields, runtime calls) still
      via `prove_arena` or malloc.

**Refs:** `docs/design.md` (Zero-Cost Abstractions), `prove-py/archive/0.9.5-PLAN.md`,
`prove-py/archive/0.9.7-PLAN.md`, `c_emitter.py:595,619`, `prove_region.c`

## 4. AI-Resistance Features — Partially Enforced

Parsing for all module-level AI-resistance keywords (`domain`, `temporal`, `invariant_network`)
is now complete — no longer causes syntax highlighting issues or spurious linting errors.
Enforcement varies by feature (see below). The `module_features_demo` example may still be
expected-to-fail pending fuller enforcement.

### 4a. Counterfactual Annotations (`why_not`, `chosen`)

- [x] Lexed and parsed into `FunctionDef.why_not` (list) and `FunctionDef.chosen` (str)
- [x] Formatter preserves them
- [x] LSP provides keyword completion with docs
- [ ] **No semantic checking** — not verified against code, not used by prover
- [ ] **No documentation generation** — not surfaced in `prove check` output

**Refs:** `docs/ai-resistance.md` (Parsed section), `parser.py:492–496`

### 4b. Temporal Effect Ordering (`temporal:`)

- [x] Parsed in module declaration (`parser.py:1234–1242`); parses arrow chains (`"a -> b -> c"`)
      into `ModuleDecl.temporal` as `list[str]`; formatter preserves correctly
- [x] **W390 enforcement** — checker now stores `_temporal_order` from `ModuleDecl` and
      `_check_temporal_ordering()` in `_check_contracts.py` warns when function bodies call
      temporal operations in wrong order

**Refs:** `docs/syntax.md` (temporal), `docs/ai-resistance.md` (Parsed section),
`_check_contracts.py:_check_temporal_ordering`

### 4c. Intent Annotations (`intent:`)

- [x] Parsed into `FunctionDef.intent` (str)
- [x] Formatter preserves it
- [~] **Warning only** — W311 if `intent` declared without `ensures`/`requires`
- [ ] **No deeper verification** — intent not compared to actual behavior

**Refs:** `docs/syntax.md` (intent), `_check_contracts.py:168–176`

### 4d. Invariant Networks (`invariant_network`, `satisfies`)

- [x] `invariant_network` blocks parsed with rules (`parser.py:1324`); parsing now complete
- [x] `satisfies` parsed on function definitions
- [x] Checker validates `satisfies` references an existing type (E382)
- [x] **Constraint type-checking** — `_check_invariant_constraints()` in `_check_contracts.py`
      type-checks constraint expressions (E396 for non-Boolean constraint type), and bare
      function references are resolved against the function table
- [x] **W391 enforcement** — warns when `satisfies` declared but no `ensures` clause exists
      (invariant satisfaction cannot be verified without postconditions)
- [ ] **No formal verification** — constraint expressions validated syntactically/type-wise
      but not proven against function implementations; no cross-function coherence checking

**Refs:** `docs/syntax.md` (invariant_network), `docs/ai-resistance.md`,
`_check_contracts.py:_check_invariant_constraints`

### 4e. Domain Declarations (`domain`)

- [x] Keyword lexed and parsed; parsing complete
- [x] **Domain profiles enforced** — `domains.py` defines `DomainProfile` / `DOMAIN_PROFILES`;
      `checker.py:_check_domain_profiles()` enforces domain-specific warnings W340–W342
- [x] **Coherence checking** — `checker.py:_check_coherence()` checks vocabulary consistency;
      `--coherence` and `--challenges` CLI flags added
- [ ] **Context-dependent syntax** — domain does not yet alter syntax or restrict constructs
      beyond warning-level enforcement

**Refs:** `docs/ai-resistance.md`, `domains.py`, `checker.py:_check_domain_profiles()`

## 5. Optimizer Passes — Documented but Not Implemented

The `docs/compiler.md` and `archive/0.5-PLAN.md` describe 6 optimizer passes. Only some
are in the actual `optimizer.py`.

**Implemented passes (in `optimizer.py:optimize()`):**
- [x] Runtime dependency collection
- [x] Tail call optimization
- [x] Dead branch elimination
- [x] Compile-time evaluation of pure calls
- [x] Small function inlining
- [x] Dead code elimination
- [x] Memoization candidate identification
- [x] Match compilation (consecutive match merge)

**Newly implemented passes:**
- [x] **Iterator fusion** — `map(filter(...))`, `filter(map(...))`, `map(map(...))` fused
      into single pass (`optimizer.py:_iterator_fusion()`)
- [x] **Copy elision** — liveness analysis avoids copies when source not reused
      (`optimizer.py:_copy_elision()`)
- [x] **Match to jump tables** — integer match patterns compiled to C `switch` statements
      (`_emit_stmts.py`)

**Refs:** `docs/compiler.md` (Optimizer Passes), `prove-py/archive/0.5-PLAN.md`,
`optimizer.py:107–116`

## 6. Concurrency / Parallelism

## 10. Contract / Verification Gaps

- [x] `ensures` / `requires` parsed, type-checked, generate property tests
- [x] `explain` blocks with strict/loose mode, named entries, `when` conditions
- [x] `believe` requires `ensures` (E393), generates 3x adversarial tests
- [x] `know` / `assume` parsed and type-checked; `assume` emits runtime assertion
- [x] `terminates` required for recursive functions (E366)
- [x] `near_miss` generates boundary tests
- [x] Mutation testing (`mutator.py`)
- [~] **Formal proof** — `ClaimProver` in `prover.py` now proves constant folding and algebraic
      identities (x==x, x-x==0, refinement constraints); emits E355/E356 (provably false) and
      W327 (indeterminate). No SMT solver; does not prove general program properties.
- [ ] **Verification chain propagation** — docs say `ensures` propagates through call graph
      and warns when called functions lack `ensures`. Current implementation: `prove check`
      reports statistics but does NOT emit per-call-site warnings for unverified callees.
- [ ] **`near_miss` distinct boundary verification** — docs say "compiler verifies each
      exercises a distinct boundary". Implementation generates test cases but doesn't verify
      distinctness.

**Refs:** `docs/contracts.md`, `docs/syntax.md`, `prover.py`, `testing.py`

## Summary: Critical V1.0 Blockers

These are the features that V1.0 claims to include (per `docs/roadmap.md`) but are missing or incomplete:

| Feature | Status | Priority |
|---------|--------|----------|
| Comptime execution | ✓ `comptime` now valid in expression position everywhere | Resolved |
| Linear types (return-position moves) | ✓ Return-position Own identifiers now tracked as moved | Resolved |
| Memory regions (temporaries via region) | ✓ String + list literals now region-allocated; `_in_return_position` guard prevents dangling pointers | Resolved |
| Temporal enforcement | ✓ W390 warning when operations called out of declared order | Resolved |
| Invariant_network enforcement | ✓ Constraint type-checking + W391 when no ensures; no formal proof | Partially resolved |
| Formal verification (`know` proofs) | Lightweight `ClaimProver` added; general proofs not possible | Low |
| Concurrency (`par_map`) | Runtime + EffectType scaffolding exists; not callable from user code | Low (may defer to post-1.0) |

**Resolved since 2026-03-08:** Iterator fusion, copy elision, jump tables, C runtime tests (4 modules),
`prove export` CLI, domain enforcement, static refinement checking for literals, complex/field-path move tracking.

**Resolved since 2026-03-09 (session 2):** Return-position moves (checker.py), temporal ordering enforcement (W390,
`_check_contracts.py`), invariant network constraint type-checking + W391 enforcement (`_check_contracts.py`).

**Resolved since 2026-03-09 (session 3):** Memory regions for string/list temporaries — `prove_string_from_cstr_region`
added to runtime; `_in_return_position` guard prevents dangling-pointer bug on return values; string and list
literals use region allocation inside functions. Comptime blocks in function bodies — one-line parser fix
(`_parse_prefix` now handles `TokenKind.COMPTIME`); emitter already worked.
