# Prove v0.9.6 — Mutation Testing

## Status: ✓ IMPLEMENTED

## Goal

Implement the `--mutate` flag for `prove build` and `prove test`. The compiler
generates mutants (small code modifications like flipping operators, removing
branches) and runs the contract-based test suite against each mutant. Surviving
mutants indicate gaps in contracts.

## Scope

- Mutant generation from AST (operator swaps, branch removal, constant changes)
- Run test harness against each mutant
- Report mutation score and surviving mutants
- Suggest contracts that would kill surviving mutants

## Additional Optimization

- **Strip unused runtime libs before compile** — Implemented in
  `c_runtime.py`. Analyzes which runtime functions are used (via regex on
  generated C code) and uses AST-level tracking (via `RuntimeDeps` in optimizer)
  to exclude unused runtime libs. Enabled by default.

## Implementation Details

### Mutation Engine (mutator.py)

- Created `Mutator` class that generates AST mutations:
  - Operator swaps: `+ → -`, `* → /`, `== → !=`, etc.
  - Constant mutations: `0 → 1`, `true → false`
  - Condition negation: `< → >=`, `!` removal
- Created `run_mutation_tests()` function that orchestrates:
  - Generating mutants from each module
  - Running tests against each mutant
  - Computing mutation score
  - Reporting surviving mutants with suggestions

### CLI Integration

- Added mutation testing to `prove build --mutate`
- Parses all .prv files, generates mutants, runs tests
- Reports mutation score and surviving mutants
- Suggests adding contracts to kill surviving mutants

### Runtime Lib Stripping

- Added `_RUNTIME_FUNCTIONS` map in c_runtime.py
- Added `_extract_function_calls()` to analyze C code
- Added `strip_unused` parameter to `copy_runtime()`
- Infrastructure in place, can be enabled when heuristic improves
