# Prove v0.9 — Lexer Export Tool

## Goal

Add a `prove export` command that generates syntax highlighting definitions
for tree-sitter, Pygments, and Chroma from the compiler's canonical token
and type lists. This eliminates keyword drift between the compiler and the
three companion lexer projects.

Current state: tree-sitter-prove, pygments-prove, and chroma-lexer-prove all
still reference `proof` (removed in v0.8.2). The `saves` verb (removed in
v0.4) has already been cleaned from all three, but the `proof` → `explain`
migration was never propagated.

## Phase 2: Sentinel Comments

Add sentinel comments to all three target files so the export tool knows
which sections to replace. This is a one-time manual edit.

### Format

```
// PROVE-EXPORT-BEGIN: <category>
... generated content ...
// PROVE-EXPORT-END: <category>
```

Categories: `verbs`, `keywords`, `contract-keywords`, `ai-keywords`,
`builtin-types`, `operators`, `literals`.

### tree-sitter-prove/grammar.js

Wrap the `verb` choice, keyword rules, contract keyword rules, AI keyword
rules, and operator rules with sentinels.

### pygments-prove/pygments_prove/__init__.py

Wrap each `words(...)` group and operator regex with sentinels. Use `#`
comment syntax:

```python
# PROVE-EXPORT-BEGIN: verbs
words(("transforms", "inputs", ...), ...),
# PROVE-EXPORT-END: verbs
```

### chroma-lexer-prove/prove/lexer.go

Wrap each regex alternation line with sentinels. Use `//` comment syntax:

```go
// PROVE-EXPORT-BEGIN: verbs
{`\b(transforms|inputs|...)\b`, chroma.KeywordDeclaration, nil},
// PROVE-EXPORT-END: verbs
```

### tree-sitter-prove/queries/highlights.scm

Wrap each keyword group with sentinels. Use `;` comment syntax:

```scheme
; PROVE-EXPORT-BEGIN: contract-keywords
[ "ensures" "requires" "explain" "terminates" ] @keyword.control
; PROVE-EXPORT-END: contract-keywords
```

## Phase 3: Export Tool Implementation

### New file: `src/prove/export.py`

#### Canonical Data Sources

| Category | Source file | Source symbol | Example tokens |
|----------|-----------|--------------|----------------|
| Verbs | `tokens.py` | `KEYWORDS` (verb kind) | transforms, validates, reads, creates, matches, inputs, outputs |
| Core keywords | `tokens.py` | `KEYWORDS` (keyword kind) | from, type, is, as, match, module, domain, ... |
| Contract keywords | `tokens.py` | `KEYWORDS` (contract kind) | ensures, requires, explain, when, terminates, trusted |
| AI-resistance keywords | `tokens.py` | `KEYWORDS` (AI kind) | why_not, chosen, near_miss, know, assume, believe, ... |
| Built-in types | `types.py` | `BUILTINS` | Integer, String, Boolean, Decimal, Float, Character, Byte, Unit |
| Generic types | `checker.py` | `_BUILTIN_TYPE_NAMES` | List, Option, Result, Error, Table |
| Built-in functions | `checker.py` | `_BUILTIN_FUNCTIONS` | len, map, filter, reduce, println, print, readln, ... |
| Operators | `tokens.py` | `TokenKind` operator variants | +, -, *, /, ==, !=, <, >, <=, >=, &&, \|\|, \|>, =>, .. |
| Boolean literals | hardcoded | — | true, false |

#### Architecture

```python
def read_canonical_lists() -> dict[str, list[str]]:
    """Read all keyword/type/operator lists from compiler source."""
    # Import from tokens, types, checker
    # Categorize KEYWORDS by their TokenKind group
    # Return dict with keys: verbs, keywords, contract_keywords,
    #   ai_keywords, builtin_types, generic_types, builtin_functions,
    #   operators, literals

def replace_sentinel_section(content: str, category: str, replacement: str) -> str:
    """Replace content between PROVE-EXPORT-BEGIN/END markers."""

def generate_treesitter(lists: dict, workspace: Path) -> None:
    """Write grammar.js and highlights.scm keyword sections."""

def generate_pygments(lists: dict, workspace: Path) -> None:
    """Write __init__.py keyword/type/operator sections."""

def generate_chroma(lists: dict, workspace: Path) -> None:
    """Write lexer.go keyword/type/operator sections."""
```

Each generator:
1. Reads the target file
2. Finds sentinel comment pairs for each category
3. Replaces the content between sentinels with generated content
4. Writes the file back

#### Workspace Discovery

The export tool assumes companion projects are siblings of the prove-py
directory:

```
/workspace/
  prove-py/           ← compiler (this project)
  tree-sitter-prove/  ← target
  pygments-prove/     ← target
  chroma-lexer-prove/ ← target
```

If a target directory doesn't exist, skip it with a warning.

### CLI Registration

Add to `cli.py`:

```python
@main.group()
def export():
    """Export syntax highlighting definitions from canonical token lists."""

@export.command()
@click.option("--build", is_flag=True, help="Also run build steps after generating.")
def treesitter(build):
    """Generate tree-sitter grammar keywords."""

@export.command()
@click.option("--build", is_flag=True, help="Also run build steps after generating.")
def pygments(build):
    """Generate Pygments lexer keywords."""

@export.command()
@click.option("--build", is_flag=True, help="Also run build steps after generating.")
def chroma(build):
    """Generate Chroma lexer keywords."""

@export.command("all")
@click.option("--build", is_flag=True, help="Also run build steps after generating.")
def all_targets(build):
    """Generate all syntax highlighting definitions."""
```

### `--build` Flag Behavior

| Target | Without `--build` | With `--build` |
|--------|-------------------|----------------|
| tree-sitter | Write `grammar.js` + `highlights.scm` | + `tree-sitter generate` + `tree-sitter test` |
| pygments | Write `__init__.py` | + `pip install -e .` (in pygments-prove dir) |
| chroma | Write `lexer.go` | + `go build ./...` (in chroma-lexer-prove dir) |

Build steps use `subprocess.run` with `cwd` set to the target directory.
Build failures are reported but don't prevent other targets from generating.
