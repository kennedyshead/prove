# Prove v0.9.7 — Ownership Complete + Memory Regions

## Goal

Complete the ownership system and implement memory regions for efficient short-lived allocations.

## Status: PLANNED

## Scope

- Compiler-inferred borrows for read-only parameters
- Comprehensive use-after-move detection
- Region-based memory allocation for local temporaries

## Implementation Details

### 1. Compiler-Inferred Borrows

The compiler should automatically infer when a parameter is used in read-only mode
and apply borrow semantics without explicit annotations.

**Approach:**
- Analyze function bodies to detect when parameters are only read (not mutated)
- For read-only parameter usage, automatically infer borrow semantics
- Apply this to parameters passed to other functions

**Files to modify:**
- `src/prove/checker.py` — add borrow inference pass
- `src/prove/types.py` — add borrow type representation

### 2. Comprehensive Use-After-Move Detection

Currently only tracks moves through function calls with Own parameters. Extend to:
- Assignment moves (`x = y` where y is moved)
- Complex expressions (chained operations)
- Nested moves (fields of owned values)

**Current state (checker.py):**
- `_moved_vars` set tracks moved variables
- `_track_moved_args()` marks moves from Own params (line 2293)
- Only tracks function call argument moves

**Files to modify:**
- `src/prove/checker.py` — extend `_track_moved_args()`, add assignment tracking

### 3. Region-Based Memory Allocation

Implement efficient allocation for short-lived values that don't escape their scope.

**Runtime changes:**
1. Add global `_global_region` to prove_runtime.c:
   ```c
   static ProveRegion *_global_region = NULL;
   ```

2. Initialize in `prove_runtime_init()`:
   ```c
   _global_region = prove_region_new();
   ```

3. Cleanup in `prove_runtime_cleanup()`:
   ```c
   prove_region_free(_global_region);
   ```

**Code generation (c_emitter.py):**
- Emit `prove_region_enter(_global_region)` at function start
- Emit `prove_region_exit(_global_region)` at function end
- Use `prove_region_alloc(_global_region, size)` for local temporaries

**Files to modify:**
- `src/prove/runtime/prove_runtime.c` — add global region
- `src/prove/c_emitter.py` — emit region enter/exit, use region alloc
