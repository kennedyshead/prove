# Prove v0.9.5 — Auto-Memoization + Memory Regions

## Status: ⚠ PARTIAL — Memoization implemented, region allocation pending

## Goal

Implement automatic memoization of pure functions and region-based memory
allocation for short-lived values. Pure functions (`transforms`, `validates`,
`reads`, `creates`, `matches`) with no side effects are candidates for
memoization. Region-based memory avoids reference counting overhead for
values that don't escape their scope.

## Scope

- Identify memoizable pure functions (small, frequently called, deterministic)
- Emit memoization tables in generated C code
- Region allocator for function-local temporaries
- Inlining of small pure functions (extend existing optimizer pass)

## Files Affected

- `src/prove/optimizer.py` — memoization candidate analysis with param_types
- `src/prove/c_emitter.py` — emit memo tables, memoized call wrapping, region includes
- `src/prove/c_runtime.py` — region allocator C runtime support (already existed)

## Implementation Details

### Memoization

- Extended `MemoizationCandidate` to store param_types for proper hash key generation
- Added `_emit_memo_tables()` to emit hash table structures for each candidate
- Added `_get_memo_key()` to generate hash keys from function arguments
- Modified function call emission to check memo cache before calling

### Memory Regions

- Added `#include "prove_region.h"` to generated C code
- Runtime region allocator exists in `prove_region.c/h`
- **NOT IMPLEMENTED**: region allocation for local temporaries in function bodies

  To implement fully:
  1. Add global `_global_region` to runtime (in prove_runtime.c)
  2. Emit `prove_region_enter()` at function start, `prove_region_exit()` at end
  3. Use `prove_region_alloc(_global_region, size)` for temporaries instead of `prove_alloc()`
