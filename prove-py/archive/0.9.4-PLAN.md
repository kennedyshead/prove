# Prove v0.9.4 — Linear Types + Ownership

## Status: ⚠ PARTIAL — Own/Mutable modifiers, use-after-move partially implemented; compiler-inferred borrows not implemented

## Goal

Implement the `Own` type modifier and compiler-inferred borrows. Functions
that take `Own` parameters consume the value — using it after the call is a
compile error. The compiler infers borrows for read-only access, reducing
annotation burden compared to Rust-style lifetimes.

## Scope

- `Own` modifier on type annotations (e.g., `File:[Own]`)
- Use-after-move detection in the checker
- Compiler-inferred borrows for read-only parameters
- `Mutable` type modifier for mutable bindings

## Files Affected

- `src/prove/types.py` — add `Own` and `Mutable` modifiers
- `src/prove/checker.py` — ownership tracking, use-after-move errors
- `src/prove/c_emitter.py` — ownership-aware memory management

## Implementation Details

### Type Modifiers (types.py)
- `has_own_modifier()` — checks if type has [Own] modifier
- `has_mutable_modifier()` — checks if type has [Mutable] modifier
- Type parsing supports `:[Modifier]` syntax (parser.py lines 649-681)

### Use-After-Move Detection (checker.py)
- `_moved_vars` set tracks moved variables
- `_track_moved_args()` — marks variables as moved when passed to Own params (line 2293)
- `_check_moved_var()` — reports E340 error for use after move (line 2300)
- **PARTIAL**: Only tracks moves through function calls with Own parameters
- **NOT IMPLEMENTED**: Assignment moves, complex expressions, nested moves

### Compiler-Inferred Borrows
- **NOT IMPLEMENTED**: Would require analyzing function bodies to detect
  read-only parameter usage and automatically inferring borrow semantics
