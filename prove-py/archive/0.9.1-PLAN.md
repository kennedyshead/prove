# Prove v0.9.1 — `[Lookup]` Type Modifier

## Summary

Replace the standalone `lookup` keyword with a `[Lookup]` type modifier on algebraic type
definitions. This integrates lookup tables into the existing type/modifier system and changes
the accessor syntax from `$` to `TypeName:value`.

## New Syntax

```prove
type TokenKind:[Lookup] is String where
    Foreign | "foreign"
    BooleanLit | "true"
               | "false"
```

### Accessor

- **Forward** (value to variant): `TokenKind:"foreign"` returns `Foreign`
- **Reverse** (variant to value): `TokenKind:Foreign` returns `"foreign"`

### Stacking

Multiple values can map to the same variant via stacking (continuation `|` lines):

```prove
    BooleanLit | "true"
               | "false"
```

Reverse lookup on a stacked variant is ambiguous and produces **E378**.

## Changes from Old Syntax

| Old | New |
|-----|-----|
| `lookup TokenKind \| String` | `type TokenKind:[Lookup] is String where` |
| `$"main"` | `TokenKind:"main"` |
| `$Main` | `TokenKind:Main` |
| One lookup per module | Multiple `[Lookup]` types per module |

## Diagnostic Codes

- **E375** — duplicate value in lookup table (unchanged)
- **E376** — lookup operand must be a literal or variant name (unchanged)
- **E377** — value or variant not found in lookup table (unchanged)
- **E378** — reverse lookup on stacked variant is ambiguous (new)

## Value Types

The value type in a `[Lookup]` definition must be `String`, `Integer`, or `Boolean`.
